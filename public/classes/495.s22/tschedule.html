CSC 495/693 – Spring 2022 – Schedule

This is a “special topics” class, and since it is not a regular class it has never been offered like this before. Because of this, the exact schedule and timing is uncertain and subject to change. This list will be adjusted as the semester progresses, and at any time the next two weeks should be pretty accurate.

After Spring Break

I’ll fill in the “after Spring Break” schedule once we see how easily we get through the first set of topics. At this point, the plan is to switch over to looking at web application vulnerabilities after Spring Break, looking at:

  • Basic web application structure and threat model
  • Web technologies and security features - JavaScript, same-origin policy, …
  • Common web application vulnerabilities - organized by the OWASP Top 10
  • Tools and Techniques: Taint analysis, the Zed Attack Proxy, and Burp Suite
  • Secure coding principles - design, testing, and project management

If time allows:

  • Advanced tools - Kali Linux and Metasploit
  • Safer language for system software development: Rust

Final Exam

Tuesday, May 3, 3:30-6:30

Topics:

  • Introduction, motivation, and terminology
  • Practices and ethics of responsible disclosure
  • Vulnerability finding - security audits, bug bounties, marketplaces

  • Recent attention: whitehouse, NIST (and consortium), OSSecurity, …

  • Resources: security notices, CVEs, exploitdb, OWASP, etc.
  • Supply chain vulnerabilites - authenticity, etc.
  • Systems/Linux review while examining some vulnerabilities
  • Memory issues - allocation, sizes, segments, buffers, I/O
  • Assembly language and machine instruction review
  • Memory safety issue 1: buffer overflows (CWE-120)
  • Memory safety issue 2: bad buffer size calculation (CWE-131)
  • Integer overflow or wrapwaround (CWE-190)
  • Protections (stackguard, aslr, nox stacks)
  • Static analysis – ideas and some existing tools
  • Symbolic execution - ideas
  • Fuzz testing - tools and infrastructure (more on this!)
  • Web applications - general structure and weak points (threat model)
  • Tools - ZAP, Burp Suite, etc.
  • Same-origin policy - cookies and javascript
  • Cross-site scripting (CWE-79)
  • Cross-site request forgery (CWE-352)
  • SQL injection (CWE-89)
  • Command injection (CWE-78)
