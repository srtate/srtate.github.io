CSC 495/693 - Assignment 4 - Due Tues, April 12

This homework refers to the “fake bank” web application that we have used in class for examples. Information on how to access the application is not included on this public web page – see the Canvas announcement for access information. The announcement also tells you how you can access the code that’s running on the server.

  1. The login code has a bad SQL injection. Exploit this vulnerability to log in to the server with username “secure”. For your assignment, report exactly what you did for this exploit (including the strings/inputs you used) and then report the balance in the “secure” account.

  2. There is a user named “hidden” on the system that is protected using measures beyond the basic bank code that you have been shown. Because of these extra protections, you will not be able to log in to this account using the SQL injection trick described in the previous question. However, you should be able to find and read a message from “hidden” to “manager” that contains a secret message. Figure out how to find and read the message, and in your assignment submission explain how you did this and what the secret message says. Note that you will not be able to see any messages sent to the “hidden” account so you will not see the XSS attempts of other students (see the next question).

  3. When the real “hidden” user logs in, they get a special cookie that is not part of a normal login session. Since you can’t log in as the “hidden” user you should have no way to access this cookie. While logged in as any other user on the system, send a message to “hidden” that performs a cross-site scripting attack to discover this cookie.
