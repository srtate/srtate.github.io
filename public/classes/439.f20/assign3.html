Assignment 3 – Complete Parser – Due Mon, Oct 26

For this assignment, you are to complete your parser to support semantic checking and syntax tree generation for the full LittleC language, including char and array types. All of the information needed regarding the language is in the Assignment 2 write-up.

If you did not get Assignment 2 finished, or would just like to get some ideas from looking at a solution, I will post my Assignment 2 solution on Tuesday, Oct 19, after the late deadline for Assignment 2 has passed.

The starter repository that is cloned when you accept the assignment invitation will include new tests for the additional language features, named “good2…” and “bad2…”. As in the two previous assignments, you are encouraged to create your own tests to share with the class, and for sharing we’ll use the FullParserTests repository

I am also providing an additional resource with this assignment: A working solution that you can run your test cases through to see what is expected. This is available on prdile.uncg.edu if you have set up your account to include the csc439 tools (see the “Option 1” document in the “Compiler Project” section of Canvas, under “Using UNCG’s PRDILE System”). Once that is set up, the program is run by typing littlec with exactly two command-line arguments: a run “style” and an input file name. If you just type littlec you’ll get the following “help” statement that tells you what the arguments are:

Run as follows (input is from stdin unless a filename is given):

   littlec mode [optinfile]

where "mode" is one of the following:
   -lt runs LexerTest
   -pt runs ParserTest
   -st outputs the syntax tree (like pt without symbol tables)
   -pi produces intermediate code output
   -pe produces an executable

Currently, only the “-lt”, “-pt”, and “-st” options are supported, but more will be added over time.
