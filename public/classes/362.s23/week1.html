  • Demos
    • Show “Set-up Information” module - talk abt GitHub acct and VM
    • Show VirtualBox - provided file creates a VM, talk about download (size/time) and show where “Import” is. Need to do this before Thursday!
    • Show loaded VM and go through some virtual hardware requirements
      • “General/Advanced/Shared Clipboard”: Talk about isolation and value of that, but also difficult of working with. Jump down to “Shared Folders” and talk about similar things there. While not required, I encourage you to keep both of these sharing options disabled and keep the VM as an isolated environment. All classwork will be transfers in and out of VM via GitHub.
      • System/RAM: 1GB is probably OK for programming tasks, but more gives better in-VM caching, esp important when running a web browser in VM; mentally subtract RAM amount from host, and think that is what your host is working with, so on a 4GB system, a 2GB VM is a big hit. Default is 3GB - can adjust (up or down) if needed. Only change when system has been shut down.
      • System/CPUs: Set for 2 – needed for some multithreading, but can dial back or up; unlike mem, CPUs are shared so not “taken away” from host
      • Storage: Virtual disks - entire disk (vdi) in one or more files; can insert and remove disks (show on optical drive)
      • Network: This is complicated, because it controls how the VM communicates with the outside world - is it part of your local network? Is it isolated? Is there a virtual network of multiple VMs? Default setting - NAT - acts roughly the same way a computer/device on a home network would act, behind your WiFi/router where in this case the router is in the host<->VM interface.
