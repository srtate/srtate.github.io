Thursday

Slides – shoot for 25 min:

  • Basic model - inputs, processing, outputs - where from/to?
  • File descriptors - numbers - streams get/put byte by byte
  • Redirection
    • Common to think of stdin/out as “console” but that’s just the default
    • Files (redirection)
    • Other programs (piping)
  • Environment variables
    • Note: note environmentAL variables
    • Think “session level” or “user level” rather than per run options
    • Mention .profile

Demos:

  • Anything “left over” from last time (there was a lot to cover!)?
  • Examining environment variables
    • Use “echo” for a few specifics like $HOME, $USER, etc.
    • Use “env” to show all
    • First pipe example: env piped to “less”
    • Then env piped to grep (look for “PATH”)
      • Then beginning with PATH (‘^PATH’)
    • Exported vs non-exported variables (echo for PS1 and env w/grep)
    • Setting environment variables (then export)
  • Redirection
    • env to file then edit file
    • echo or “cat -” with redirection for quick and dirty file creation
  • New commands: head, tail, sort, uniq (incl with -c), cut
    • What are first few lines of book.txt? (less, scroll back, editor, then head)
    • last few lines? use tail
    • See a few more of the last lines: tail -25
    • How many lines total (use wc)
    • Look at /etc/passwd
      • First few lines?
      • usernames are in arbitrary order - use “sort” to put in order
      • first few? Pipe sort into head – then last few (pipe to tail)
    • Interested in shells that accounts use
      • Talk about fields - count out to shell (7th field)
      • cut to look at just 7th field
      • then uniq to get rid of adjacent lines that are duplicates
      • then sort first to put all duplicates together
      • what if we want stats like how many use each shell? uniq -c
  • Archives and compression: tar, gzip, bzip2, xz
    • subdir has three files - can we bundle those to transfer as 1 file?
    • very common for distributing datasets, programs, etc.
    • use tar cf all.tar subdir
    • tar tf to see contents
    • then create new dir t, cd into t, and tar xf ../all.tar
    • look at size of all.tar (about 10k)
    • compress with gzip and look again
    • untar with tar xzf
    • copy book.txt to c1 c2 and c3 and apply gzip, bzip2, and xz to each
